# Water: Data Schemas & Flow

## Purpose
Define data structures and defaults that flow through the system. All rami use these schemas to ensure consistency.

## File
`schemas.js`

## Schemas Defined

### Message
```javascript
{
  id: 'string',              // Unique ID (timestamp-based)
  user: 'string',            // Username (or username)
  text: 'string',            // Message content
  job: 'string',             // Job name (or job_id)
  timestamp: 'string',       // ISO timestamp
  subcategory: 'string',     // 'Seeking Solution' or Chat-pad name
  individualChat: 'string|null', // Individual chat name (or individual_chat_id)
  isProblem: 'boolean'       // Whether message is a problem
}
```

### User
```javascript
{
  username: 'string',
  lastSeen: 'string'         // ISO timestamp
}
```

### Job Subcategory
```javascript
{
  job: 'string',
  subcategory: 'string'      // Chat-pad name
}
```

### Individual Chat
```javascript
{
  job: 'string',
  subcategory: 'string',     // Factory Order name
  chatName: 'string',
  id: 'string'               // Unique ID (timestamp-based)
}
```

### Database
```javascript
{
  users: [],                 // Array of User objects
  messages: [],              // Array of Message objects
  jobSubcategories: [],      // Array of JobSubcategory objects
  individualChats: []        // Array of IndividualChat objects
}
```

## Field Aliases
Supports legacy and new naming conventions:
- `user` ↔ `username`
- `job` ↔ `job_id`
- `individualChat` ↔ `individual_chat_id`

All rami accept either naming convention via SAP normalization.

## Usage
```javascript
const { schemas, defaults } = require('./water/schemas');
const newDB = { ...defaults.database };
const jobs = [...defaults.predefinedJobs];
```
