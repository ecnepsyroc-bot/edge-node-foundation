# Ramus: Chat

## Purpose
Message persistence, subcategory management, and chat storage/retrieval.

## Responsibilities
- Load/save chat database (JSON file)
- Migrate legacy database formats
- Store and retrieve messages
- Manage job subcategories (Factory Orders)
- Manage individual chats within Factory Orders
- Register users
- Delete messages (by ID, by user, by subcategory, by individual chat)
- Track message metadata (problems, timestamps)

## Public Interface

### Constructor
```javascript
new ChatManager(dbFilePath)
```

### Methods
- `registerUser(username)` - Register or update user last seen time
- `addMessage(msgData)` - Add message to database (uses SAP sanitization)
- `getMessages(filters)` - Get messages with optional filters (job, subcategory, individualChat)
- `deleteMessageById(messageId)` - Delete single message by ID
- `deleteMessagesByUser(username, job, subcategory)` - Delete user's messages (excludes problems)
- `createJobSubcategory(job, subcategory)` - Create Factory Order
- `deleteJobSubcategory(job, subcategory)` - Delete Factory Order and its messages
- `getJobSubcategories(job)` - Get all Factory Orders for a job
- `createIndividualChat(job, subcategory, chatName)` - Create individual chat
- `deleteIndividualChat(job, subcategory, chatName)` - Delete individual chat and its messages
- `getIndividualChats(job, subcategory)` - Get all individual chats for a Factory Order

## Dependencies
- Node.js `fs` module
- Node.js `path` module
- `water/schemas.js` (data structure defaults)
- `sap/validators.js` (sanitization)

## Does NOT Handle
- Job validation (uses JobsManager via graft)
- HTTP/WebSocket communication (uses ServerManager via graft)
- NLP processing
- UI rendering

## Integration Points
- Connected to ServerManager via `server-chat` graft
- Connected to JobsManager via `chat-jobs` graft
- Uses WATER schemas for data structure
- Uses SAP validators for sanitization

## Database Format
```json
{
  "users": [{ "username": "string", "lastSeen": "ISO timestamp" }],
  "messages": [{ "id": "string", "user": "string", "text": "string", ... }],
  "jobSubcategories": [{ "job": "string", "subcategory": "string" }],
  "individualChats": [{ "job": "string", "subcategory": "string", "chatName": "string", "id": "string" }]
}
```

## Field Normalization
Supports both legacy and new field names:
- `user` / `username`
- `job` / `job_id`
- `individualChat` / `individual_chat_id`
