# Graft: Chat ↔ Jobs

## Purpose
Connects ChatManager (persistence) to JobsManager (validation).

## Bridges
- `ChatManager` (rami/chat/manager.js)
- `JobsManager` (rami/jobs/manager.js)

## Responsibilities
- Validate jobs exist before creating messages/subcategories/chats
- Validate subcategory names (Factory Orders)
- Validate individual chat names
- Wrap ChatManager methods with JobsManager validation

## Public Interface

### Methods
- `addMessage(msgData)` - Add message with job validation
- `createSubcategory(job, subcategory)` - Create Factory Order with validation
- `createIndividualChat(job, subcategory, chatName)` - Create individual chat with validation

## Validation Flow
1. Check job exists via `jobsManager.jobExists()`
2. Validate name via `jobsManager.validateSubcategoryName()` or `validateChatName()`
3. If valid, delegate to `chatManager` method
4. If invalid, throw error

## Does NOT Handle
- HTTP/WebSocket communication (delegated to server-chat graft)
- Message storage logic (delegated to ChatManager)
- NLP processing

## Integration Pattern
```javascript
const chatJobsGraft = new ChatJobsGraft(chatManager, jobsManager);
// Used by orchestrator for validated operations
chatJobsGraft.createSubcategory(job, subcategory);
```

## Anti-Corruption Layer
Prevents invalid data from entering ChatManager:
- No nonexistent jobs
- No invalid subcategory names
- No invalid chat names
