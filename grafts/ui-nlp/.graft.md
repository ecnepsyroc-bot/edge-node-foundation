# Graft: UI ↔ NLP

## Purpose
Provides NLP processing to client-side UI via HTTP endpoint.

## Bridges
- `NLPProcessor` (rami/nlp/processor.js)
- Client UI (rami/ui/public/app.js)

## Responsibilities
- Expose NLP processing as HTTP API
- Accept text input from client
- Return color-coded segments, HTML, and analysis
- Handle errors gracefully

## HTTP Endpoint

### Process Text
```
POST /api/nlp/process
Body: { "text": "message text" }
Response: {
  "original": "message text",
  "segments": [{ "text": "word", "color": "#4CAF50", "pos": "noun" }],
  "html": "<span style='color:#4CAF50'>word</span>...",
  "analysis": { "wordCount": 5, "nounCount": 2, ... }
}
```

## Public Interface

### Methods
- `processMessage(text)` - Process text and return all NLP data
- `handleNLPRequest(req, res, serverManager)` - Handle HTTP endpoint

## Does NOT Handle
- Message storage
- User authentication
- Job management
- WebSocket communication

## Status
Endpoint ready but not yet called by UI. NLP color-coding can be integrated into `displayMessage()` function in app.js when ready.

## Future Integration
To activate NLP in UI:
1. Call `/api/nlp/process` when displaying messages
2. Replace plain text with color-coded HTML
3. Cache results to avoid repeated processing
