# Sap: Validation & Guardrails

## Purpose
Validate and sanitize all data flowing through the system. Prevents malformed data from corrupting rami.

## File
`validators.js`

## Validation Functions

### Text Sanitization
- `sanitizeText(text)` - Remove control characters, trim, limit to 5000 chars
- `validateUsername(username)` - Check username valid (1-50 chars)
- `validateJobName(jobName)` - Check job name valid (1-100 chars)

### Message Validation
- `validateMessage(msg)` - Validate complete message object structure
- `sanitizeMessage(msgData)` - Sanitize and normalize message fields
  - Accepts both `user` and `username`
  - Accepts both `job` and `job_id`
  - Accepts both `individualChat` and `individual_chat_id`
  - Auto-detects `isProblem` from "🔴 Seeking Solution:" prefix
  - Preserves alias fields for backward compatibility

### Subcategory Validation
- `validateSubcategory(name)` - Check Factory Order name valid
- `sanitizeJobSubcategory(data)` - Sanitize subcategory object

### Individual Chat Validation
- `validateChatName(name)` - Check individual chat name valid (1-100 chars)
- `sanitizeIndividualChat(data)` - Sanitize individual chat object

## Guardrails

### Input Limits
- Username: 1-50 characters
- Job name: 1-100 characters
- Chat name: 1-100 characters
- Message text: 5000 characters max
- Control characters: Removed (except newlines/tabs)

### Field Normalization
Ensures consistent internal representation:
```javascript
// Input can be either format
{ user: "Cory" } or { username: "Cory" }
// Output preserves both
{ user: "Cory", username: "Cory" }
```

### Auto-Detection
- `isProblem`: Automatically set to `true` if text starts with "🔴 Seeking Solution:"
- Timestamp: Auto-generated if missing
- ID: Auto-generated if missing

## Integration
All rami use SAP validators:
- `ChatManager.addMessage()` calls `sanitizeMessage()`
- `JobsManager.addJob()` calls `validateJobName()`
- `createSubcategory()` calls `validateSubcategory()`
- `createIndividualChat()` calls `validateChatName()`

## Does NOT Handle
- Business logic (in rami)
- Storage (in ChatManager)
- HTTP routing (in grafts)
- UI rendering (in UI branch)

## Purpose in Tree
SAP wraps existing logic with safety checks. It doesn't rewrite business rules; it guards the boundaries.
